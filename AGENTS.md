# AGENTS.md

## Cursor Cloud specific instructions

### Overview

BuFlow is a Fortran 90 CFD (Computational Fluid Dynamics) solver with automatic differentiation (AD) capabilities for aerodynamic shape optimization. The code is generated by TAPENADE (INRIA) from a Julia prototype (FvCFD.jl).

### System dependencies

- **gfortran** (>= 13): Fortran compiler
- **liblapack-dev**, **libblas-dev**: LAPACK/BLAS linear algebra libraries

These must be installed via `apt-get` (handled by the update script).

### Build

Compile order matters due to Fortran module dependencies:

```bash
gfortran -c -O2 TypesModule_d.f90
gfortran -c -O2 meshdeformationn_d.f90
gfortran -c -O2 BuFlow_test_d.f90
gfortran -c -O2 run_parameter_d.f90
gfortran -O2 -o buflow TypesModule_d.o meshdeformationn_d.o BuFlow_test_d.o run_parameter_d.o -llapack -lblas
```

### Running

```bash
ulimit -s unlimited   # required: large stack for mesh arrays
./buflow
```

The solver reads the OpenFOAM mesh from `mesh/OFairfoilMesh/` and CSV data files (`wing.csv`, `boundary_inoutput_xy.csv`). It runs ~1635 time steps to simulate transonic airfoil flow at Mach 0.786.

### Key caveats

- **No Makefile or build system**: compilation must be done manually with gfortran in the dependency order above.
- **Stack size**: must set `ulimit -s unlimited` before running or the solver will segfault on large mesh arrays.
- **No automated tests**: the codebase has no test suite. Verification is done by running the solver and checking output.
- **No linter**: Fortran code does not use a linter. Compilation with `-fcheck=all` can catch runtime issues.
- **NaN in solution**: the solver currently produces NaN values during the simulation; this is a pre-existing numerical stability issue in the codebase, not an environment problem.
- **Code has commented-out allocations**: many global allocatable arrays in `TypesModule_d.f90` have their `ALLOCATE` calls commented out in the source. Fixes were applied to restore critical allocations.
